# Engineering Management Choices

## Why Choose pipx + uv

Based on the project's actual setup, choosing the **pipx + uv** combination has the following advantages:

### 1. **Advantages of uv**

- **Ultra-fast package manager**: uv is a Python package installer and resolver written in Rust, 10-100x faster than pip
- **All-in-one solution**: uv integrates virtual environment management, dependency resolution, package installation, and more
- **Lockfile support**: provides deterministic dependency locking through `uv.lock` (similar to npm's `package-lock.json`)
- **Modern dependency management**: supports `dependency-groups` (see lines 13-23 in `pyproject.toml`) to better organize development and production dependencies
- **Built-in tool execution**: `uv run` can execute commands directly in the virtual environment without manual activation

### 2. **Advantages of pipx**

- **Isolated tool installation**: pipx is designed for installing and managing Python CLI tools (such as uv itself), avoiding pollution of project dependencies
- **Globally available**: installed tools run in dedicated virtual environments and can be used anywhere
- **Tool version management**: makes it easy to manage versions of different tools without conflicting with project dependencies

### 3. **Best Practices for Using Them Together**

```bash
# Install uv with pipx (global tool)
pipx install uv

# Manage project dependencies with uv (project level)
uv sync --group dev    # Install development dependencies
uv run pytest          # Run tests
uv run ruff check      # Run lint checks
```

### 4. **How This Appears in the Project**

The `Makefile` shows extensive use of uv commands:

- `uv sync`: synchronize dependencies (lines 85-86)
- `uv lock`: lock dependency versions (lines 96 and 105)
- `uv run`: run various tools (pytest, ruff, etc.)
- `uv build/publish`: build and publish packages

This combination provides **clear separation of responsibilities**: pipx manages global tools, while uv manages project dependencies. It keeps the development environment clean and delivers an excellent dependency management experience.

## Choice of githook + lefthook

- **githook**: Git's native hook mechanism, suitable for simple pre-commit checks, but requires manual setup and management
- **lefthook**: a modern Git hook management tool that provides richer features and a better user experience, supports multi-language hook scripts, and allows unified hook management via configuration files
